作成した app.py を動かすサーバーを ip アドレス 10.133.2.200 に用意した。サーバーにアップロー ドして動かしてみよう。

1 サーバーへのリモートログイン

user{出席番号+50} というアカウント作ってあるのでログインしてみよう。コマンドラインで以下を入力して サーバーにログインしてみよう (user出席番号のアカウントは別なクラスが使っています)。

出席番号 4 番の人の場合 

ssh user54@10.133.2.200

出席番号 12 番の人の場合は 

ssh user62@10.133.2.200

パスワードは全員共通で password0322 と設定してある。 以下の例では出席番号 4 番の人の場合で説明していくが、各自自分の出席番号に置き換えて読んでいくこと。 ログインすると 

user54@jikkenserver

と表示されてログインできたことが確認できる。念の為に 

ip a

とコマンドラインに入力して 10.133.2.200 のサーバーにログインできているか確認しよう。ログイン できることを確認したら 

exit

と入力してログアウトしよう。以下のように表示されたらログアウトできている。 

Connection to 10.133.2.200  closed
exit

2 サーバーへのファイル転送

作成した app.py があるディレクトに cd コマンドで移動しよう。次に以下のように入力すると app.py サーバーに送ることができる。 

rsync app.py user54@10.133.2.200:~/

user54 の部分は各自自分のアカウント名に置き換えること。 パスワードを入力するように言われるので入力する。 ファイルを送ることに成功した場合は何も表示されない。 タイプミスなどで失敗した場合はエラーが表示されるので、内容を確認して再度行う。

次に説明した 「サーバーへのリモートログイン」の手順でログインし ls コマンドでファイルがサーバーに転送されていることを確認する。 次のコマンドで転送先のファイルの中身が見れる。 

more app.py

間違いなく自分の作ったファイルであるか確認しよう（先輩の作ったファイルが残っていることがある）。

3 サーバーでのファイル編集 (emacs)

一般的に、遠方にあるサーバーをリモート操作する場合は windows で使えるような便利な機能は使えない。 このときによく使われるのは emacs または vim またはnanoというエディタが使われる。 本授業では emacs を使う。 以下のコマンドを入力して app.py を修正してみよう 

emacs app.py

emacs の使い方は以下にわかりやすく書かれている。次のリンク先の操作方法を読んで app.py を編集してみよう。 https://emacs-jp.github.io/tips/emacs-in-2020

– Ctrl キーと x キーを同時に押してから次に Ctrl キーと c キーを同時に押すと emacs を終了で きる。 – Ctrl キーと g キーを同時に押すとほとんどの動作をキャンセルして前の状態に戻れるのが重要な 点なので覚えておこう。 – この資料では GUI emacs の説明をしている. リモートで動いているのは CUI 版であるが、使い方は同じである。

またこのリンクの資料 http://www.f.waseda.jp/moriya/im6/UNIX/Emacs.pdf では日本語の入力方法についても解説している。 日本語入力は基本的に Ctrl キーと \ キーを同時に押すと日本語入力モードになる。 emacs を使って app.py の最後の方にある 

app.run ( )

(10050+自分の出席番号）のポートを使うように次のように変更してみよう。 出席番号 4 番の場合は 

app.run (host=’0.0.0.0’,port=10054)

app.run (host=’0.0.0.0’,port=10062)

のように修正する。修正したら 

python3 app.py

としてウェブサービスを実行してからブラウザで http://10.133.2.200:10054/ を開いてみよう。10054 の部分は(10050+自分の出席番号)の数字に置き換えること。 他人のサービスも見れるので(10050+ 出席番号)のところを変えて見てみよう。

4 ログインしないウェブアプリの実行

サーバーにログインして 

python3 app.py

を実行してウェブアプリを実行すると、ログイン中しかウェブアプリが動いてくれない。ログインしていない 時も実行しっぱなしにするには以下のようにする。 

nohup python3 app.py &

最後の& を忘れないこと。これを実行してから 

exit

でログアウトしてもウェブアプリが動いたままになっていることを http://10.133.2.200:10050+自分の出席番号/

をfirefoxなどのWebのブラウザで開いて確認してみよう。

5 ウェブアプリの停止

以下で自分の動かしているウェブアプリのジョブ番号が探せる。 

ps aux | grep python

一覧で表示されるジョブ一覧で自分のものを確認して、例えばジョブ番号が 12345 だったら以下でウェブア プリが停止できる。 

kill 12345

http://10.133.2.200:10050+自分の出席番号/

をfirefoxなどのWebのブラウザで開いて停止しているか確認してみよう。
